AI Football Commentary

When a user uploads a video of a FIFA football match, the system processes it in several steps:

  1. The input video is first processed by the YOLOv8 model, which annotates the raw video with bounding boxes and labels for all detected objects.
  2. The annotated video is then passed to the GPT-4 Vision model, which generates a detailed commentary based on the annotations. YOLOv8 ensures accurate labeling, making it easier for GPT-4 Vision to produce context-rich commentary such as accurate player names, the ball, the field, the socres of each team, etc.
  3. Once the commentary script is generated, it is passed to a Text-to-Speech (TTS-1) model to convert the text into audio.
  4. The final step combines the audio commentary with the original video to create the output video, which is displayed to the user.

The entire process is hosted on a Flask web app and made accessible via ngrok for public use. Users can also download the final video if they wish.

YOLOv8 Model

The YOLOv8 model is trained on 38 classes, covering all aspects of a football game. Key details include:

  • The dataset, consisting of over 6000 images, is self-created and self-annotated due to the poor quality of open-source data.
  • The model achieves an impressive mAP of 86.8%, with a precision of 85.7% and a recall of 82.9%.
  • At the time of development, no other models performed as well as ours for this specific use case.

Dementia Wandering Prevention System

The system monitors CCTV footage to detect and prevent dementia patients from wandering beyond designated areas. Here's how it works:

  1. When someone passes in front of a CCTV camera, the system records the video and processes it frame-by-frame.
  2. Each frame is analyzed by our custom vision model to identify individuals.
  3. If the person is not a registered patient, the system takes no action.
  4. If a patient is detected alone:
    • The system saves a frame of the patient to Azure Blob Storage
    • An SMS alert is sent to the nurse's phone via Sinch SMS API
    • The alert includes a message about the boundary breach and the stored image
  5. If the patient is accompanied by a nurse, no alerts or images are stored.

Technical Implementation

  • Uses Custom Object Detection Model trained on a dataset of over 3000 images with 2 classes (nurse and patient) using General [A1] for accurate person detection
  • Processes video at 30 FPS for real-time monitoring. However only the first and the last frame of the video are processed and stored to speed up processing time by the model
  • Integrated with Azure Blob Storage for secure image retention
  • Uses Sinch SMS API for immediate nurse notifications

Model Performance

  • mAP of 99.9% for patient detection
  • Precision of 99.3% (low false positives)
  • Recall of 100.00% (minimizes missed detections)

Kubernetes Image Classification System

A distributed computer vision system deployed on Kubernetes that classifies user-uploaded images into four vehicle categories with 98.9% accuracy.

  1. Users upload images through a UI (automobile, airplane, ship, or truck)
  2. The UI pod forwards images to the model inference pod via Kubernetes services
  3. The fine-tuned Xception model processes the image with the following workflow:
    • Image preprocessing (resizing, normalization)
    • Feature extraction through 71 deep convolutional layers
    • Classification with softmax activation
  4. Results are returned to the UI with:
    • Predicted class label
    • The Predicated Image

System Architecture

  • 4 dedicated pods: UI frontend, Data preprocessing, Model training, Model inference
  • Inter-pod communication via Kubernetes ClusterIP services
  • Shared a single Persistent volume

Resume Classification using BERT

An intelligent resume classification system that leverages BERT (Bidirectional Encoder Representations from Transformers) to automatically categorize resumes into 25 different job categories with high accuracy.

  1. Data Preprocessing:
    • Text cleaning and normalization
    • Special character removal and ASCII conversion
    • Custom word tokenization for technical terms (e.g., "C++")
    • Stop word removal and lemmatization using spaCy
    • URL removal and gibberish text filtering
  2. BERT Model Architecture:
    • Pre-trained BERT base model (12 layers, 768 hidden size)
    • Fine-tuned for sequence classification with 25 output classes
    • Attention mechanism with 12 heads and dropout regularization
    • Maximum sequence length of 128 tokens
  3. Training Process:
    • 80-20 train-test split with stratification
    • AdamW optimizer with learning rate 1e-5
    • Batch size of 32 for efficient training
    • 100 epochs with early stopping

Technical Implementation

  • Built using PyTorch and Hugging Face Transformers
  • Preprocessing pipeline using NLTK, spaCy, and custom regex patterns
  • Data visualization with Matplotlib and Seaborn

Model Performance

  • Validation Accuracy: 80%
  • Matthews Correlation Coefficient: 0.72

IoT Weather Station & Urban Farming

Developed an integrated IoT solution combining Arduino-based sensor networks with Raspberry Pi data processing and Blynk cloud visualization for precision urban farming and weather monitoring.

System Architecture

The system collects environmental data through distributed sensors, processes it through multiple hardware layers, and visualizes it through a cloud dashboard:

  1. Input Sensors (Arduino Layer):
    • DHT11 - Temperature and humidity monitoring
    • Grove Moisture Sensor - moisture of soil
    • Grove Ultrasonic Ranger - Plant growth monitoring
    • Grove Light Sensor - Light intensity measurement
  2. Output Actuators (Control Layer):
    • Servo-operated watering
    • LCD local display module
  3. Data Processing (Raspberry Pi):
    • Serial communication with Arduino nodes
    • Data normalization and calibration
    • WiFi/Ethernet connectivity
  4. Cloud Integration (Blynk Platform):
    • Real-time dashboard visualization
    • Mobile alert notifications
    • Historical data charts

System Workflow

  1. Data Collection Phase:
    • Sensors continuously sample environmental parameters
    • Arduino nodes pre-process sensor readings and convert them into JSON format
  2. Data Transmission:
    • Sensor data is transmitted via serial communication to the Raspberry Pi gateway
    • The Raspberry Pi sends data to the Blynk cloud using WiFi
  3. Processing & Response:
    • Raspberry Pi decode the JSON data, extract values and apply conditional logics to determine environmental conditions
  4. User Interaction:
    • Real-time monitoring via Blynk mobile/web dashboards
    • Historical data analysis through exported reports

def send_to_blynk(token, pin, value):
    api_url = f"https://blynk.cloud/external/api/update?token={token}&{pin}={value}"
    response = requests.get(api_url)
    return "200" in str(response)

def main():
    # Initialize serial communication with Arduino
    ser = serial.Serial('/dev/ttyACM0', 9600, timeout=1)
    ser.reset_input_buffer()

    while True:
        if ser.in_waiting > 0:
            try:
                # Read and parse sensor data
                data = json.loads(ser.readline().decode('utf-8').rstrip())
                
                # Extract sensor values
                light = data.get('light')
                temp = data.get('temp')
                hum = data.get('hum')
                dist = data.get('dist')
                moisture = data.get('moisture')
                
                # Determine growth and weather conditions
                growth = "ripe" if dist <= 7 else "unripe"
                weather = "sunny" if light > 300 else "cloudy" if temp < 26 and hum < 90 else "rainy"
                
                # Send data to Blynk
                send_to_blynk(token, "v0", str(light))
                send_to_blynk(token, "v1", str(temp))
                send_to_blynk(token, "v2", str(hum))
                send_to_blynk(token, "v3", growth)
                send_to_blynk(token, "v4", weather)
                
                # Control servo based on moisture
                if moisture < 400:
                    servo.start(0)
                    servo.ChangeDutyCycle(5)
                    time.sleep(1)
                    servo.ChangeDutyCycle(10)
                    time.sleep(0.5)
                    servo.stop()
                else:
                    servo.stop()
                    
            except Exception as e:
                print(f"Error: {e}")

if __name__ == '__main__':
    main()
